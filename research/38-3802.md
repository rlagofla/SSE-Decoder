# 通道 (38, 3802) 分析 — **沪市 基金/LOF L2 快照 (FAST 编码)**

## 1) 流量画像

- 帧数: **2757**
- 总字节: **2,245,769**
- 平均帧长: ~815 byte
- 压缩: 部分 ZIP (zip_ok=815 / 2757 = 29.6%) — **仅大帧压缩, 小帧明文**
- 40B 外层头: `flag_28=0x08000000`, `flag_36=15`

## 2) 40B 外层头示例

```

[0x04]  00 00 08 00   length      = 2048
```

## 3) body 字节特征

- ZIP + raw-deflate (大帧) 或明文 (小帧)
- **FAST stop-bit 编码**, 结构与 **6/3202 高度相似**, 但证券段不同
- 显著 ASCII 标记:
  - `54 52 41 44 c5` = "TRADE" (市场分段)
  - 阶段码: **没有在每块里都出现 "T111"** (与 6/3202 略有差异)

## 7) 盘口+近期成交结构

第一个证券的 `f0` 系列序列:

```
f0 07 6e e3 0f d1 82 82 c0 0f d1
f0 07 6e dd 17 b9 82 81
f0 07 6e dc 17 b9 82 81
f0 07 6e d3 17 b9 82 81
f0 07 6e c3 04 78 e9 82 81
f0 07 6e bb 06 05 b9 82 81
f0 07 6e a1 07 e9 82 81
...
```

- `f0` = 0xF0 MSB=1, 档位标记 (类似 6/3202 的 `90`)
- `07 6e xx` = 价格 (3 byte FAST, 差值相近 → 连续价位)
- `xx xx xx` = 量
- `82 / 82 81` = 档位编号 / flag

这个 `f0 ...` 系列共约 10~14 条 → **10 档盘口 + 最近成交的混合表示**。

## 8) 为什么不是逐笔

同 6/3202:
- 带盘口档位序列
- 带 "TRADE" 市场分段字符串
- 每支证券一整块聚合 — 不是逐条件 delta

## 9) 结论

**(38, 3802) = 沪市 基金 / 可转债 / REITs 的 L2 完整快照 (FAST)**, 格式是 6/3202 的同族变体, 服务于 11xxxx/118xxx/113xxx 代码段。

- 体积比 6/3202 小是因为这个细分市场的证券数更少, 活跃度也较低
- **非逐笔通道**
