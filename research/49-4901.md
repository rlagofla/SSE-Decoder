# 通道 (49, 4901) 分析 — **沪市 债券 L2 聚合快照 (M201 协议, MD301 Stream)**

## 1) 流量画像

- 帧数: **2457**
- 总字节: **1,708,812**
- 平均帧长: ~695 byte
- 压缩: **几乎全部 ZIP** (zip_ok=1962 / 2457 = 79.9%)
- 40B 外层头: `flag_28=0x021faf40` (与 47/4701 族一致), `flag_36=0`

## 2) 40B 外层头示例

## 3) body 字节特征

- ZIP 解压后是 **"M201" 标签族**的聚合快照 — 与 47/4701 **同协议族**
- body[4..8] = `"M201"` (ASCII)
- 固定 49B 包头, 后接 `NoEntries` × 变长 SecurityEntry
- **与 47/4701 的差异** (决定性特征):
  - `category` 字段 = `0x00000203` (vs 47/4701 的 `0x00000103`)
  - `MDStreamID` = **"MD301"** (vs 47/4701 的 "MD002")
  - `SecurityID` = **8 位数字** (vs 47/4701 的 6 位 + 2 空格)
  - `Symbol` GBK = **8 字节全空格** (债券无中文简称)
  - `TradingPhaseCode` = **"T 01    "** (vs 47/4701 的 "T111    ")
  - MDEntryType 扩展: 在 `'2' '4' '7' '8' '0'x5 '1'x5` 之外, 追加 **`'x'` `'6'` `'z1'` `'z2'`** 等债券专用条目

## 5) 债券 SecurityEntry 解剖 (第 1 条, 偏移 49)

```

---  --------------------------------------------  ---------------------------
 0   01                                            security_type = 0x01

**关键差异**: TradingPhaseCode 是 `"T 01    "` (T + 空格 + 01 + 4 空格), 不同于股票的 "T111"。
债券使用**交易阶段代号 "01"** (连续竞价) / "T 09" (闭市) 等二字代号。

## 6) 债券 MDEntry 类型扩展

每条 MDEntry 仍是 19 byte (off 0 entry_type, off 1 pad, off 2..9 price, off 10..17 qty, off 18 pos):

标准类型 (与 47/4701 共享):
- `'2'` Last, `'4'` Open, `'7'` High, `'8'` Low
- `'0'` Bid × 5 档, `'1'` Ask × 5 档

**债券独有类型** (观测到):
- `'x'` (0x78) — 可能是 **加权平均价 / 匹配成交量**
- `'6'` (0x36) — 可能是 **债券收益率 / 基准收益率**
- `'z1'` `'z2'` (0x7A + '1' / '2') — 债券特有 —— 可能是:
  - `z1` = 应计利息 / 累积利息
  - `z2` = 净价 / 全价 差异

观察: NoMDEntries = 18 而不是 14, 多出来的 4 条就是 `'x' '6' 'z1' 'z2'`。

## 7) 证券代码范围

"10010775"

```

8 位代码与沪市**质押式回购 / 国债 / 地方债 / 企业债**编码体系一致 (100xxx 开头 + 子号)。

## 8) 结论

**(49, 4901) = 沪市 债券 L2 聚合快照**, 使用盛立 ODP **M201 协议族**同一定长结构, 但:
- MDStreamID 是 "MD301"
- category = 0x0203
- SecurityID 是 8 位数字
- TradingPhaseCode 格式是 "T xx"
- MDEntry 多 4 种债券专用类型 (x / 6 / z1 / z2), 共 18 条/证券

客户端要解析这一通道, 只需扩展现有 47/4701 的 `SecurityEntryView` 支持:
1. `MDStreamID == "MD301"` 判定
2. `NoMDEntries` 可能 > 14
3. 新增 MDEntryType desc 表项

**非逐笔通道**。和 39/3901 (债券逐笔) 形成"快照 + 逐笔" 互补。
