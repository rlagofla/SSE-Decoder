# 通道 (38, 3815) 分析 — **FAST 阶段心跳 (基金/可转债段状态帧)**

## 1) 流量画像

- 帧数: **20**
- 总字节: **1,220**
- 平均帧长: **61 byte** (固定)
- 压缩: **全明文**
- 40B 外层头: `flag_28=0x08000000`, `flag_36=15` (**与 38/3802 同族**)

## 2) 40B 外层头示例

```
[0x00]  00 04 c4 53   magic
[0x04]  00 00 00 3d   length      = 61
[0x08]  00 00 00 26   channel_hi  = 38
[0x0c]  00 00 0e e7   channel_lo  = 3815
[0x10]  00 05 a9 80   seq         = 371072
[0x14]  00 00 00 00   zero
[0x18]  4a ea c5 4b   t24
[0x1c]  08 00 00 00   flag_28
[0x20]  00 00 00 00   compressed = 0
[0x24]  00 00 00 0f   flag_36    = 15
```

## 3) body 结构 (21 byte)

```
f8 1d e7 2d 3b 61 e2 30 30 30 30 30 b0 2d 3b 61 e3 09 54 4a 93
```

### FAST 字段解析

```
off  bytes                   FAST 解码               含义
---  ----------------------  ----------------------  -----------------------
 0   f8                      120                     header_a
 1   1d e7                   (29<<7)|0x67 = 3815     template_id = 3815 = channel_lo ✓
 3   2d 3b 61 e2             u32 ≈ 97,182,562        时间戳 / 批次 id #1
 7   30 30 30 30 30 b0       "000000"                SecurityID = "000000" (占位)
13   2d 3b 61 e3             u32 ≈ 97,182,563        时间戳 / 批次 id #2 (= 前者 + 1)
17   09 54 4a 93             u32 = 20,260,115        **TradeDate = 20260115**
```

## 4) 与 6/3115 的关系

结构**几乎完全对称**:

| 维度         | 6/3115                 | 38/3815                |
|--------------|------------------------|------------------------|
| body 长度    | 20 byte                | 21 byte (多 1 字节)    |
| template_id  | 3115                   | 3815                   |
| 字段数       | 5                      | **6** (多一个时间戳)   |
| SecurityID   | "000000"               | "000000"               |
| TradeDate    | 20260115               | 20260115               |
| 关联主通道   | 6/3113 (指数)          | 38/3802 (基金/可转债)  |

**38/3815 是 38/3802 的心跳版**, 用法同 6/3115 之于 6/3113:
- 保活 / 批次同步 / 交易日翻滚

## 5) 两个时间戳字段的猜测

off 3 和 off 13 都是 4 byte FAST, 值连续相差 1:
```
off 3:  2d 3b 61 e2   → 97,182,562
off 13: 2d 3b 61 e3   → 97,182,563
```

这不是 HHMMSS (太大), 更像是:
- **通道内批次/帧 id** (前后两个 id, 一个代表当前帧, 一个代表下一帧预留)
- 或 **ApplSeqNum** 前后两条 (某种引用"前一帧 + 自身"的结构)

## 6) 结论

**(38, 3815) = 基金/可转债段阶段心跳帧**, 非行情, 非逐笔。

- 与 38/3802 组成"行情 + 心跳"对
- 每 ~10 分钟 1 帧
- 解析价值有限, 但可辅助客户端判断"通道是否在活"
