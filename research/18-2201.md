# 通道 (18, 2201) 分析 — **FIX 交易阶段状态通知 (35=h, TradingSessionStatus)**

## 1) 流量画像

- 帧数: **8** (全 pcap 最稀疏之一)
- 总字节: **7,022**
- 平均帧长: ~878 byte
- 压缩: **全明文**
- 40B 外层头: `flag_28=0x021faf90`, `flag_36=0`

## 2) 40B 外层头示例

```
[0x00]  00 04 c4 53   magic
[0x04]  00 00 00 ac   length      = 172
[0x08]  00 00 00 12   channel_hi  = 18
[0x0c]  00 00 08 99   channel_lo  = 2201
[0x10]  00 00 0d 83   seq         = 3459
[0x14]  02 1f af 90   zero
[0x18]  00 00 00 00   t24
[0x1c]  02 1f af 90   flag_28
[0x20]  00 00 00 00   compressed = 0
[0x24]  00 00 00 00   flag_36    = 0
```

## 3) body 结构 — FIX 35=h

body 是**纯文本 FIX 消息**, 132 byte:

```
8=FIXT.1.1 | 9=108 | 35=h | 49=EzEI.1.1 | 56=EzSR.201060 |
34=3460 | 52=20260114-09:53:50.991 | 347=GBK | 167=67 |
339=3 | 336=111          | 393=786 | 10=190
```

### FIX 标签表

| Tag | 含义                         | 本帧值                 |
|-----|------------------------------|------------------------|
| 8   | BeginString                  | FIXT.1.1               |
| 9   | BodyLength                   | 108                    |
| 35  | MsgType                      | **h** = TradingSessionStatus |
| 49  | SenderCompID                 | EzEI.1.1               |
| 56  | TargetCompID                 | EzSR.201060            |
| 34  | MsgSeqNum                    | 3460                   |
| 52  | SendingTime                  | 20260114-09:53:50.991  |
| 347 | MessageEncoding              | GBK                    |
| 167 | SecurityType                 | 67 (自定义, 阶段通知?)  |
| 339 | TradSesSubID                 | 3                      |
| 336 | **TradingSessionID**         | **"111     "** (8 字节右补空格) |
| 393 | **TotalNumSecurities**       | 786                    |
| 10  | CheckSum                     | 190                    |

## 4) 关键字段解读

- **tag 336 = "111"** — 与 47/4701 M201 内 TradingPhaseCode 的 "T111    " 语义一致, 指**连续竞价 (regular trading)**
- **tag 393 = 786** — 本帧通知**共涉及 786 支证券**开始某阶段
- **tag 339 = 3** — 交易会话子 ID = 3 (盛立 ODP 自定义)
- **tag 167 = 67** — SecurityType 67 (不是标准 FIX 证券类别, 是阶段通知专用)

## 5) 与其它通道的关系

本通道**不承载行情, 只广播"市场阶段切换"事件**:
- 每当市场从一个阶段切到下一个 (如开盘前 → 连续竞价 → 午休 → 尾盘集合竞价 → 闭市), 这条通道广播一帧
- 与 47/4701 的 M101 通道通知**互为镜像**:
  - 47/4701 的 M101 走二进制 48B 定长, 在行情通道内嵌入
  - 18/2201 走 FIX 明文, 独立通道
- 客户端可选择二者之一做阶段同步

|-------|--------------------------|-------------|---------|---------------------|
| 1     | 09:53:50.991             | 111         | 786     | 连续竞价中 (在样本开始) |

一天内这条通道仅广播 ~6~10 帧 (开盘前, 集合竞价, 连续竞价, 午休, 下午开, 尾盘, 闭市), 非常稀疏。

## 7) 结论

**(18, 2201) = FIX 协议的市场阶段状态通知 (TradingSessionStatus)**, 低频广播。

- **非逐笔**
- 不承载行情数据, 只通知阶段切换
- 解析成本极低 — 纯 ASCII FIX, SOH 切 tag 即可
- 是补齐"市场时钟"的参考信息, 对量化回测可用 (精确定位各阶段时间戳)
