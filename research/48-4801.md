# 通道 (48, 4801) 分析 — **沪市 期权 L2 聚合快照 (M201, EMD102 Stream)**

## 1) 流量画像

- 帧数: **105** (极低)
- 总字节: **28,330**
- 平均帧长: ~270 byte
- 压缩: 部分 ZIP (zip_ok=40/105 = 38%, 小帧明文, 大帧压缩)
- 40B 外层头: `flag_28=0x021fb0ad`, `flag_36=0`

## 2) 40B 外层头示例 (明文小帧)

```
[0x00]  00 04 c4 53   magic
[0x04]  00 00 00 5d   length      = 93
[0x08]  00 00 00 30   channel_hi  = 48
[0x0c]  00 00 12 c1   channel_lo  = 4801
[0x10]  00 00 27 07   seq         = 9991
[0x14]  02 1f b0 ad   zero
[0x18]  00 00 00 00   t24
[0x1c]  02 1f b0 ad   flag_28
[0x20]  00 00 00 00   compressed = 0
[0x24]  00 00 00 00   flag_36    = 0
```

## 3) body 结构

- 大多数帧是 **"M201" 标签的空快照 (NoEntries = 0)** — 只承载心跳 / 时间同步
- 偶尔有非空帧 (NoEntries > 0), 每条证券条目是期权合约:
  - MDStreamID = **"EMD102"** (6 字节, 比股票的 "MD002" 多一个 "E" — "E"xecutable 期权?)
  - SecurityID = **8 位数字 "02688xxx"** (个股期权合约代码)
  - Symbol GBK = **4 字 中文** (对应标的证券简称, 如"豫东生物"类)
  - **期权特有的 category** = `0x00000303`
  - TradingPhaseCode = **"A1      "** (8 字节, 期权专用阶段码)
  - 非交易时大量字段为 0

## 6) TradingPhaseCode "A1" 含义

期权合约的阶段码体系与股票/债券不同:

| 代码 | 推测含义           |
|------|--------------------|
| A1   | 未开盘 / 开盘前    |
| A2   | 集合竞价           |
| A3   | 连续竞价           |
| A4   | 暂停 / 休市        |

(需参考 SSE 个股期权业务规则确认)

## 7) 代码 "02688xxx" 解读

"02688xxx" 实际对应 **SSE 个股期权 8 位合约代码**, 头 3 位 "026" 是**期权合约编号段**, 后 5 位 "88xxx" 是**当月/季月到期月份 + 行权价序号**。

首帧证券代码:
```
"02688513"  "02688514"  "02688516"  "02688517" ...
"02688189"  "02688290"  "02688291"  "02688521" ...
"02688292"  "02688293"  "02688295"  ...
```

都在 02688xxx 段内 —— 属于同一批次发布的期权合约。

## 8) 结论

**(48, 4801) = 沪市 个股期权 L2 聚合快照 (M201 协议, EMD102 stream)**。

本 pcap 取样时期权合约多处于未开盘/无成交状态 (PreClose = 0, NoMDEntries = 0), 所以绝大多数帧是"空快照"。

采样到的证券条目告诉我们该通道**会在开盘后承载期权合约的实时行情**, 格式同 47/4701 的 M201 一致, 仅:
- security_type = 0x00 (不是 0x01)
- MDStreamID 首字节 "E" (Executable / Electronic 推测)
- TradingPhaseCode 采用 "Ax" 编码

**非逐笔通道**。
